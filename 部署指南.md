## 第一步：创建 GitHub 仓库

> GitHub 是一个代码托管平台，免费用户创建的仓库都是公开的（别人能看到代码，但看不到你的评分数据，因为数据在程序运行后才会生成）。

### 1.1 登录 GitHub

打开 https://github.com/login 登录你的账号。

### 1.2 创建新仓库

1. 点击右上角 **+** 号 → **New repository**
2. 按如下填写：
   - **Repository name**（仓库名称）：`judge-scoring-app`
   - **Description**（描述，选填）：`裁判评分系统 - 答辩组/实操组在线打分`
   - **Public 或 Private**：选 **Public**（免费套餐只能用 Public，这不影响数据安全）
   - **不要勾选** "Add a README file"、"Add .gitignore"、"Choose a license"
3. 点击 **Create repository**

> ⚠️ 创建完成后，你会进入一个页面，里面显示了几行命令。**保持这个页面不要关**，下一步要用。

## 第二步：将本地代码推送到 GitHub

### 2.1 打开终端

在 Windows 中打开 **Git Bash**（如果安装了 Git）或使用 **VS Code 的终端**。

> 💡 **快捷方式**：在 `judge-scoring-app` 文件夹里右键 → "Open Git Bash here"

### 2.2 执行推送命令

在终端中依次输入以下命令（逐行复制粘贴，每输入一行按回车）：

```bash
cd /c/Users/lx/Desktop/judge-scoring-app
```

```bash
git init
```

```bash
git add .
```

```bash
git commit -m "init: 裁判评分系统"
```

```bash
git branch -M main
```

```bash
git remote add origin https://github.com/你的GitHub用户名/judge-scoring-app.git
```

> ⚠️ 把上面命令中的 `你的GitHub用户名` 换成你真实的 GitHub 用户名
>
> 例如：`git remote add origin https://github.com/zhangsan/judge-scoring-app.git`

```bash
git push -u origin main
```

> 🔑 执行这步时，会弹出一个 GitHub 登录窗口：
> 1. **第一次使用**：选择 "Sign in with your browser" 或在终端输入用户名和密码
> 2. **现在 GitHub 用 Token 代替密码**：如果提示输入密码，请用 Personal Access Token
>    - 获取 Token：https://github.com/settings/tokens → Generate new token (classic) → 勾选 `repo` → Generate → **复制并保存好这个 token**
>    - 在密码提示框里粘贴这个 token（粘贴后不会显示任何字符，是正常的）

### 2.3 验证推送成功

推送完成后，回到浏览器中 GitHub 的那个页面，刷新一下（按 F5）。你应该能看到代码文件都出现在仓库里了。

> ✅ **成功标志**：GitHub 上能看到 `app.py`、`utils/`、`requirements.txt` 等文件

## 第三步：部署到 Streamlit Community Cloud

> Streamlit Community Cloud 是 Streamlit 官方提供的**免费托管服务**，直接把你的 GitHub 仓库部署成网站，自动生成公网链接。

### 3.1 登录 Streamlit Community Cloud

1. 打开 https://share.streamlit.io/
2. 点击 **Sign in with GitHub**
3. 授权 Streamlit 访问你的 GitHub 账号（点击 "Authorize streamlit"）

### 3.2 部署应用

1. 登录成功后，点击右上角的 **New app** 按钮

2. 按如下填写：
   - **Repository**（仓库）：选择 `你的用户名/judge-scoring-app`
   - **Branch**（分支）：`main`
   - **Main file**（主文件）：`app.py`
   - **App URL**（会自动生成）：可以保持默认，也可以自定义（如 `judge-scoring`）

3. 点击 **Deploy!**

### 3.3 等待部署完成

- 部署过程大约 **1-3 分钟**
- 页面会显示部署日志（显示安装依赖等过程）
- 当显示 **"Your app is ready!"** 时，部署完成

### 3.4 获取公网链接

部署完成后，你会看到类似这样的地址：

```
https://你的用户名-judge-scoring-app-随机字符.streamlit.app/
```

**这就是裁判们需要访问的地址！** 复制这个链接。

## ❓ 常见问题

### Q: 页面加载很慢？
Streamlit Cloud 免费套餐在长时间无人访问后会进入"休眠"。下次访问时需要约 15 秒冷启动。这是正常的。

### Q: 评分数据会丢失吗？
数据保存在服务器的文件系统中。**免费套餐的服务器偶尔会重置，导致数据丢失。** 建议：
- 管理员**定期导出 Excel**（在管理页面操作）
- 每天评分结束后导出一次备份

### Q: 登录后看到的还是登录页？
尝试清除浏览器缓存，或重新打开链接。

### Q: 如何修改管理密码？
编辑 `app.py`，找到 `password == "zpds2026"` 这行，把 `zpds2026` 改成你想要的密码，然后重新部署。

### Q: 如何修改评分标准？
编辑 `utils/scoring.py`，修改 `DEFENSE_CRITERIA`（答辩组）或 `PRACTICAL_CRITERIA`（实操组）中的各项分数，然后重新部署。

### Q: 所有裁判的评分数据是共享的吗？
是的！所有裁判使用同一个链接，评分数据保存在同一个地方。管理员可以导出包含所有裁判评分的 Excel。
