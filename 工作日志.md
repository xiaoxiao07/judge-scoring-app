## 二、裁判分组（3组）

### 1️⃣ 线上实操 — 总分 70 分
- **评分方式：** 数字输入
- **特色：** ⏱ 用时输入 + 扣分项（内联+勾选）+ 否决项（勾选总分归零）

| 模块 | 评分项 | 分值 |
|------|--------|:---:|
| 一、大模型智能体解析(30分) | 大模型推理过程展示 | 6 |
| | 任务一：场景语义解析 | 12 |
| | 任务二：指令提取与步骤输出 | 12（内联扣分：每步一半） |
| 二、VM视觉流程与数据协议(20分) | 视觉流程运行状态 | 8 |
| | 固定数据格式与数值输出 | 12（内联扣分：每步一半） |
| 三、机器人仿真动作与闭环(20分) | 数据接收与代码响应 | 8 |
| | 初始位姿偏移与复位 | 12（内联扣分：每步一半） |

**扣分项（8项）：**
| 扣分项 | 处罚 |
|-------|:---:|
| 大模型界面只显示最终指令 | 扣除6分 |
| VM软件通讯界面无数据 | 扣除8分 |
| 机器人日志区无数据 | 扣除8分 |
| 任务2指令提取顺序错误 | 每步扣除一半分数 |
| VM发送数据格式不符 | 每步扣除一半分数 |
| 机械臂未按规定位姿 | 每步扣除一半分数 |
| 机器人仿真动画不直观 | 扣除8分 |
| 屏幕共享未设置时钟 | 扣除10分 |

**否决项：** 日志区无数据/硬编码伪装、播放预设轨迹动画 → 勾选后总分归零

## 五、部署说明

### 部署方式
- 推送 GitHub → Streamlit Cloud 自动部署
- GitHub 仓库：https://github.com/xiaoxiao07/judge-scoring-app

### ⚠️ 重要注意事项

1. **数据丢失问题：** 每次 `git push` 部署新版本，Streamlit Cloud 会重建容器，`data/` 目录下的评分数据会丢失。需要在管理页面定期导出 Excel 备份。

2. **git push 网络问题：** 有时连不上 github.com（端口 443 被阻），需要 VPN 或代理（如 `http://127.0.0.1:7890`）

3. **推送命令：**
   ```bash
   git add .
   git commit -m "修改内容说明"
   git config --global http.proxy http://127.0.0.1:7890
   git config --global https.proxy http://127.0.0.1:7890
   git push
   git config --global --unset http.proxy
   git config --global --unset https.proxy
   ```
